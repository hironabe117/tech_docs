# 1. Node.js
- Node.jsは、JavaScriptの実行環境である。
- 従来はWebブラウザ上で動作していたJavaScriptを、サーバサイドでも実行できるようにしたプラットフォームであり、次のような特徴がある。
  - サーバサイドでJavaScriptを実行可能にする環境
  - npmを通じて豊富なパッケージエコシステムにアクセス可能
  - モジュールベースの開発をサポート
  - クロスプラットフォームで動作（Windows,macOS,Linux）
  - 高速な非同期I/O処理を実現

- 従来のJavaScriptが主にWebブラウザ上で動作するのに対し、Node.jsを用いることでJavaScriptをサーバ上でも実行できる。
- これにより、JavaScriptを用いたWebアプリケーションの開発が、バックエンドからフロントエンドまで一貫して可能となる。
- 近年では、BunやDenoなどのより新しいJavaScriptのランタイム環境の活用も広がっている。
- Node.jsは、npm（Node Package Manager）を通じて豊富なライブラリやツールを利用できるため、開発効率が非常に高いのが特徴である。
- 公式サイトは[こちら](https://nodejs.org/ja)

# 2. TypeScript
- TypeScriptは、JavaScriptのスーパーセット（上位互換）であり、JavaScriptのすべての機能に加えて静的型付けを追加した言語である。
- モダンなWebアプリケーション開発において、TypeScriptが提供する型安全性と開発効率の向上が重要な利点となる。
- TypeScriptで書かれたコードは直接実行されるのではなく、トランスパイルという変換処理を経て、JavaScriptのコードに変換された上で実行される。 
  - node@23.6.0からTypeScriptでの実行が正式サポートされているらしい。が、依然としてブラウザではサポートされていない状態の模様。）
- トランスパイル時に「型チェック」が行われ、型の不整合があればコンパイルエラーとして即座に検出される。
## 2-1. TypeScriptの主なメリット
- バグの早期発見：コンパイル時に型の不一致や潜在的なエラーを検出できる。
- コードの自己文書化：型定義によってコードの意図が明確になる。
- IDEのサポート向上：コード補完や入力候補の表示が正確になる。
- リファクタリングの安全性：変更による影響範囲を型システムがチェックする。
- チーム開発の効率化：大規模プロジェクトでのコミュニケーションコストが減少する。
